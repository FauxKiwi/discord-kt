pruneMembers

suspend fun pruneMembers(days: Int, reason: String?, computePruneCount: Boolean = true, roles: List<Snowflake>? = null): Int
Content copied to clipboard

Prunes the guild from its inactive members.

The inactive members are denoted if they have not logged on in days number of days and they have no roles.

You must have the kick_members permission to use this.

To check how many members you would prune without actually pruning, see the estimatePrunedMembers() function.

To prune members that have specific roles see the roles parameter.

Return

The number of members pruned. If computePruneCount is false then this returns 0.

Parameters

days

The number of days before counting as inactive.

reason

The reason for doing this action. Shows up on the audit log.

computePruneCount

Whether to compute the prune count. This defaults to true which makes it prone to timeouts in very large guilds. In order to prevent timeouts, you must set this to false. If this is set to false, then this function will always return 0.

roles

A list of abc.Snowflake that represent roles to include in the pruning process. If a member has a role that is not specified, they’ll be excluded.

Throws

discord.exceptions.Forbidden

You do not have permissions to prune members.

discord.exceptions.HTTPException

An error occurred while pruning members.
